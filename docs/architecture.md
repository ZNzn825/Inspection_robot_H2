# 系统架构 / System Architecture

## 整体架构

本系统是一个面向加氢站安全巡检的自主机器人平台，由以下子系统组成：

```
┌──────────────────────────────────────────────────────┐
│                   Qt 上位机 (mainwindow)               │
│                                                        │
│  ┌─────────┐ ┌──────────┐ ┌────────┐ ┌───────────┐  │
│  │ AGV控制  │ │ 机械臂控制│ │ 传感器  │ │ 视频检测   │  │
│  │ CAN总线  │ │ TCP/IP   │ │ 串口   │ │ ZED SDK   │  │
│  └────┬─────┘ └────┬─────┘ └───┬────┘ └─────┬─────┘  │
├───────┼─────────────┼──────────┼─────────────┼───────┤
│       │             │          │             │        │
│  ┌────┴────┐  ┌─────┴─────┐ ┌─┴───────┐ ┌───┴──────┐ │
│  │ AGV底盘  │  │AUBO i16  │ │ H₂+火焰  │ │ ZED 2    │ │
│  │ IICT6   │  │ 机械臂    │ │ 传感器   │ │ 双目相机  │ │
│  └─────────┘  └───────────┘ └──────────┘ └──────────┘ │

└──────────────────────────────────────────────────────┘
```

## 数据流

1. **LiDAR** (Livox MID360) 采集点云 → Fast-LIO 实时定位与建图 → 发布里程计/位姿
2. **上位机** 订阅 ROS 位姿 → 巡检任务逻辑 → 发送导航目标 (move_base actionlib)
3. **AGV** 通过 CAN 总线执行运动指令（前进/后退/转向）
4. **到达巡检点** → 机械臂执行预设检测动作 → 传感器采集数据
5. **ZED 相机** 实时视频流 → HSV 目标检测 → 异常判定
6. **红外热像仪** 温度数据 → 温度异常报警

## 软件架构关键设计

### 主窗口 (mainwindow.cpp ~4100行)
单体 QMainWindow 类，集成所有子系统控制。通过定时器驱动周期性任务：
- AGV 自动巡检序列
- 传感器数据轮询
- 视频帧处理
- ROS 状态更新
- 电池电量监测

### ROS 进程管理
通过 QProcess 管理 6 个 ROS 节点进程（radar, localization, movebase, navigation, agv, initpos），每个进程在 ROS 工作空间中独立启动。

### 传感器数据线程
独立的 `DataThread` 处理串口传感器数据采集（H₂ + 火焰），避免阻塞 UI。

### 视频处理线程
`VideoProcessingThread` 继承 QThread，处理 ZED 相机视频流和目标检测，通过信号槽机制将处理结果发送到 UI 线程显示。
